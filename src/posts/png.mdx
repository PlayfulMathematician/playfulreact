# The Anatomy of A PNG

At the start of a PNG is series of magic bytes, this series is around 8 bytes. After the magic bytes is a series of chunks. A chunk has 4 components, stored as follows:

- The size of the actual contents of the chunk (network order)
- The type of the chunk (4 bytes)
- The actual contents of the chunk
- CRC-32 checksum of the second and third components of the chunk.
  There are 3 chunk types we care about, IHDR, IEND, and IDAT. Let's talk about IHDR first.

# IHDR

IHDR is, as the name suggests, a header for the images. The first 8 bytes, as used to store the width and height, of course as Big Endian. The next 5 bytes stores a bunch of more specific stuff. I hard-coded, those last 5 bytes. But for the sake of completeness, let me describe them now.

- The first byte describes the bit depth, I chose to have each color channel have 8 bits of precision
- The second byte describes color type, the number 6 corresponds to RGBA color type
- The third byte describes the compression method, as there is only one compression method (compression method 0), I choose this method
- The fourth byte describes the filter method, again I chose 0, the only available option
- The fifth byte describes interlacing, I wasn't sure what interlacing was when I wrote this code, so I just set it to zero, or no interlacing.
  Here is the corresponding code, I added comments after the fact.

```cpp
void write_ihdr(const Image &img, std::ostream &stream) {
    std::vector<unsigned char> buffer; // a buffer to store the ihdr contents
    std::vector<unsigned char> width_buffer = u32_be(img.width); // a buffer to store the big endian image width
    std::vector<unsigned char> height_buffer = u32_be(img.height); // a buffer to store the big endian image height
    for (unsigned char c : width_buffer) {
      buffer.push_back(c); // write width to buffer byte by byte
    }
    for (unsigned char c : height_buffer) {
      buffer.push_back(c); // write height to buffer byte by byte
    }
    buffer.push_back(0x08); // 8 bit color depth
    buffer.push_back(0x06); // RGBA
    buffer.push_back(0x00); // the only available option
    buffer.push_back(0x00); // the only available option
    buffer.push_back(0x00); // no interlacing
    write_chunk(buffer, "IHDR", stream); // write chunk
  }
```

# IEND

This chunk is by far the simplest, the contents of the chunk are empty. This chunk simply must appear at the end. I am not gonna include the code because it is simply not interesting

# IDAT

This chunk is where things get complicated. So instead of explaining the chunk bit by bit. Let's try to understand how we go from an image to this chunk. Let's talk about filtering

# Filtering

An image is made up long horizontal "scanlines", these go left to right, each pixel within these scanlines can be stored as 4 bytes in RGBA format. Scanlines are stored, top to bottom. Now there exists an incredibly important part of the PNG format called filtering. We take each scanline and apply a function to each of the bytes, this function makes it considerably more compressable. The function we use depends on the scanline and the function is prepended to the start of each scanline in the form of a filter byte. I do not care about compression though, so I will not apply any filtering. Although, it would be more accurate to say we apply the NONE filter to each scanline. This is the code that filters.

```cpp
std::vector<unsigned char> filtered; // create a buffer to store the filtered version
for (uint32_t i = 0; i < img.height; i++) { // iterate through scanlines
  filtered.push_back(0x00); // add filter byte (0x00 for NONE)
  for (uint32_t j = 0; j < 4 * img.width; j++) { // go through each byte of the scanline
    filtered.push_back(img.rgba[4 * img.width * i + j]); // simply push it
  }
}
```

# RFC-1950

I mentioned compression, so how does that work? Well, it uses RFC-1950, the next step in this process is to take the filtered image and compress it. The RFC-1950 format has an incredibly simple format. There are 2 bytes at the start, CMF, and FLG. Followed of course by the compressed data, followed by an Adler-32 checksum. The CMF and FLG is 0x78 and 0x01 respectively. The compressed content is compressed with a different method

# RFC-1951

RFC-1951 is a schema for compression, it splits everything into blocks, the first byte explains the block. It contains 3 bits that we care about, 2 of which describe the type of the block and the other 1 is used to determine if we are at the last block. I don't want to do compression as that seems difficult, so we are choosing block type 0. Now, blocktype zero starts off with the length of the data followed by the bit negation of the length of the data. The length is stored in 2 bytes, this means that each block can only stored 65535 bytes, this is why we need multiple blocks.

To review, we split up the filtered png file into groups of 65535 bytes. We prepend 0x00, if we are not at the end and prepend 0x01, if we are. Afterwords we prepend the length and it's bit negation. We do this to every single block. This is the compressed RFC-1951.

Here is the corresponding code:

```cpp
  std::vector<unsigned char> buffer;  // buffer to store the data
  buffer.push_back(0x78); // CMF
  buffer.push_back(0x01); // FLG
  size_t offset = 0; // how far into the filtered
  size_t remaining = filtered.size(); // how far left
  // repeat until no more left
  while (remaining > 0) {
    // if we are more than 65535, max out
    uint16_t chunk_len =
        remaining > 65535 ? 65535 static_cast<uint16_t>(remaining);
    // are we at the last block
    bool final_block = (remaining <= 65535);
    // if so push 0x01, otherwise don't
    buffer.push_back(final_block ? 0x01 : 0x00);
    // this is the length
    uint16_t len = chunk_len;
    // it's bit negation
    uint16_t nlen = ~len;
    // write length big endianly
    buffer.push_back(len & 0xFF);
    buffer.push_back((len >> 8) & 0xFF);
    // write negation big endianly
    buffer.push_back(nlen & 0xFF);
    buffer.push_back((nlen >> 8) & 0xFF);
    // write the image
    buffer.insert(buffer.end(), filtered.begin() + offset,
                  filtered.begin() + offset + chunk_len);
    // we handled the chunk account for it
    offset += chunk_len;
    remaining -= chunk_len;
  }
```

Now, remember that check sum at the end of RFC 1950, let's tack that on there too.

```cpp
  // get checksum
  uint32_t chk = adler32(filtered);
  for (char c : u32_be(chk)) {
    buffer.push_back(c); // write it
  }
  write_chunk(buffer, "IDAT", stream); // write chunk
```

# Putting it all together

We can combine everything in the following function:

```cpp
  void write_png(const Image &img, std::ostream &stream) { // taking an image and a stream
    write_header(stream); // writing magic bytes
    write_ihdr(img, stream); // write ihdr
    write_idat(img, stream); // write idat
    write_iend(stream); // write iend
  }
```

We then finally produce the image:

```cpp

  int main() {
    Image img; // the img object
    img.width = 1024; // width
    img.height = 1024; // height

    img.rgba = std::vector<uint8_t>(); // initialize the vector
    for (uint32_t y = 0; y < img.height; y++) { // iterate through scanlines
      for (uint32_t x = 0; x < img.width; x++) { // for each scan line write a pixel
        // this is how we get the pixel
        img.rgba.push_back(x / 4);
        img.rgba.push_back(y / 4);
        img.rgba.push_back(x / 8 + y / 8);
        img.rgba.push_back(255);
      }
    }
    std::ofstream file("image.png", std::ios::binary); // creating an ostream
    write_png(img, file); // writing the png to the ostream
  }
```

And THAT is how we produced the image above.
